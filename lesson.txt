الأوامر الأساسية
git init
: يستخدم لإنشاء مستودع Git جديد في الدليل الحالي. 
git clone <url
: يقوم بإنشاء نسخة محلية من مستودع Git موجود عن طريق استنساخ رابط URL الخاص به [1، 2].
git add <file/./
: يقوم بإضافة التغييرات في ملفات معينة أو جميع الملفات (باستخدام git add .) إلى منطقة التجميع (Staging Area) استعدادًا للإلتزام [1، 5].
git commit -m "رسالة توضيحية"
: يقوم بحفظ التغييرات التي تم إدراجها في منطقة التجميع إلى سجل الإصدارات، مع تضمين رسالة توصف هذه التغييرات [1، 5].
git status
: يعرض حالة المستودع الحالي، مثل الملفات التي تم تعديلها، وتلك التي تم تتبعها، والملفات التي تنتظر الحفظ في الـ commit التالي [1، 5].
git log
: يعرض سجل الإصدارات (commits) مع تفاصيل كل commit، مثل تاريخه وكاتبه ورسالته [1، 3].
git diff
: يعرض الاختلافات بين الإصدارات المختلفة من الملفات، مما يساعد على رؤية التغييرات المحددة التي تمت على كل سطر [1، 5].
أوامر إدارة الفروع
git branch
: يستخدم لعرض قائمة بالفروع الموجودة حاليًا وإنشاء فروع جديدة. 
git checkout <branch_name
: يسمح لك بالانتقال بين الفروع المختلفة في المستودع الخاص بك [1، 3].
git merge <branch_name
: يدمج تغييرات فرع معين في الفرع الحالي الذي تعمل عليه. 
أوامر التفاعل مع المستودعات البعيدة
git remote -v
: يعرض قائمة بالمستودعات البعيدة المرتبطة بالمستودع المحلي الخاص بك. 
git push
: يقوم بدفع التغييرات من المستودع المحلي إلى المستودع البعيد (مثل GitHub أو GitLab) [1، 5].
git pull
: يقوم بجلب آخر التغييرات من المستودع البعيد ودمجها في المستودع المحلي.
